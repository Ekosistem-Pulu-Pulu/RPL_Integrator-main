# STANDAR DOKUMEN DESAIN — API GATEWAY / INTEGRATOR
**Skema Tugas Besar Mata Kuliah RPL 2**
Dosen: M. Yusril Helmi Setyawan, S.Kom., M.Kom.

---
Kelompok API Gateway / Integrator wajib menyusun dokumen akhir dengan bagian-bagian berikut:

1. **Deskripsi Aplikasi**: Menjelaskan tujuan Gateway, peran dalam ekosistem, dan stakeholder.
2. **Use Case / Fitur Utama**: Daftar fitur utama (Routing, Validasi, Logging).
3. **Diagram Arsitektur**: Diagram sederhana yang menunjukkan Gateway sebagai titik kumpul alur sistem (menghubungkan node-node).
4. **Flow Proses (IPO)**: Menjelaskan Input-Proses-Output tiap fitur utama Gateway.
5. **API Endpoint**: Daftar lengkap endpoint API (method, URL, request, response) sebagai standarisasi komunikasi antar sistem.
6. **Integrasi SmartBank**: Menjelaskan bagaimana Gateway meneruskan request ke SmartBank secara aman.
7. **Desain Database**: Tabel utama + relasi sederhana (misal: tabel untuk log/history request).
8. **Mekanisme Transaksi**: Alur pemotongan *Fee Gateway*.
9. **UI Sederhana**: (Opsional untuk Gateway, mungkin dashboard log sederhana).
10. **Skenario Pengujian**: Test case validasi token (berhasil/gagal) dan routing.
11. **Kendala & Solusi**: Masalah yang dihadapi saat routing/integrasi.
12. **Dokumentasi Tim**: Pembagian tugas anggota kelompok.
